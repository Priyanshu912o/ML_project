# Machine Learning Project: Multi-Exposure Image Enhancement & Evaluation

## Intent Behind This Project
This project explores the impact of a synthetic multi-exposure image enhancement technique on the performance of state-of-the-art computer vision models. Specifically, it applies Mertens fusion with synthetic exposures and noise injection to images of cats and dogs, and evaluates how this enhancement affects:
1. **Feature Extraction Efficiency**: Using Vision Transformers (ViT) to compare the token count, processing speed, and cosine similarity of generated features.
2. **Classification Accuracy**: Training Logistic Regression and SVM models on the extracted ViT features to see if the enhancement improves or degrades classification performance for the dataset.
3. **Segmentation Consistency**: Evaluating whether the object boundaries in the enhanced images are preserved effectively using a Transformer-based segmentation model (SegFormer).

## How It Works
1. **Image Enhancement**: The pipeline reads images from the `original/` dataset. For each image, it creates multiple synthetic exposures by applying random gamma corrections, exposure factors, and noise. These exposures are merged using Mertens fusion, downsampled (pixel binning), and saved to the `enhanced/` folder.
2. **Feature Extraction**: The script uses a pre-trained `timm` Vision Transformer (`vit_base_patch16_224`) to extract the CLS token embeddings (768-D) from both the original and enhanced images. The features are explicitly saved as `.npy` files.
3. **Classification**: Logistic Regression and SVM models are trained on the `.npy` feature vectors to classify cats vs. dogs. The accuracy of the models on original vs. enhanced features is subsequently compared.
4. **Segmentation Testing**: A pre-trained SegFormer model calculates the Intersection over Union (IoU) between the segmentation masks of original and enhanced images to ensure object structures remain intact throughout the transformation.

## Structure of Dataset
The dataset within the project is organized as follows:
```text
dataset/
├── original/       # Contains raw input images
│   ├── cats/       # Original cat images
│   └── dogs/       # Original dog images
├── enhanced/       # Auto-generated by the project
│   ├── cats/       # Enhanced cat images
│   └── dogs/       # Enhanced dog images
└── features/       # Auto-generated .npy feature vectors
    ├── original/   # Features extracted from original images
    │   ├── cats/
    │   └── dogs/
    └── enhanced/   # Features extracted from enhanced images
        ├── cats/
        └── dogs/
```

## Cleaning Up (`clean_stuff.sh`)
The project includes a cleanup utility script: `clean_stuff.sh`. 
This script is used to conveniently delete all generated files inside the `dataset/enhanced` and `dataset/features` directories while keeping the core directory structure fully intact. It is extremely useful when you want to reset the project state, clear up disk space, or re-run the pipeline from scratch without losing the core `original` dataset images.

**To run the clean script:**
```bash
./clean_stuff.sh
```
